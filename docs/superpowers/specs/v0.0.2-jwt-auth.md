# SPEC: Autenticação JWT Simples para API REST

**Milestone:** v0.0.2  
**Created:** 2026-05-27  
**Stack:** Next.js 16 + Prisma + PostgreSQL + jose

---

## 1. Contexto

### 1.1 Projeto
- **Nome:** cabala-dos-caminhos
- **Stack:** Next.js 16 App Router, Prisma ORM, PostgreSQL
- **Usuários existentes:** Já existe modelo `User` com email e passwordHash

### 1.2 Objetivo
Implementar autenticação JWT simples para proteger rotas da API REST existentes.

### 1.3 Escopo
- ✅ Login com email/senha
- ✅ Middleware para proteger rotas
- ✅ Helper para verificar JWT em rotas API
- ✅ Logout (clear cookie)
- ❌ Registro de novos usuários (já existe flow via Supabase)
- ❌ Refresh tokens
- ❌ OAuth/Social login

## 2. Decisões de Design (Auto)

| Decisão | Escolha | Rationale |
|---------|---------|-----------|
| Biblioteca JWT | **jose** | Funciona em Edge Runtime (middleware), zero dependencies, Web Crypto API |
| Algoritmo | **HS256** | Simétrico, simples para single-server. RS256 overkill para MVP |
| Armazenamento token | **HttpOnly cookie** | XSS protection (localStorage vulnerável) |
| Expiração | **24 horas** | Balance segurança/usabilidade |
| Validação | **Middleware (coarse) + Helpers (fine)** | Edge-first, depois re-validação em rotas que precisam |

## 3. Arquitetura

```
┌─────────────────────────────────────────────────────────────────┐
│                     REQUEST FLOW                                 │
│                                                                  │
│  Client Request                                                  │
│       │                                                          │
│       ▼                                                          │
│  ┌─────────────┐     Protected Route?                          │
│  │  Middleware │ ──► NÃO ──► Continue                           │
│  │  (Edge)     │                                                │
│  └──────┬──────┘                                                │
│         │ YES                                                   │
│         ▼                                                        │
│  Extract + Verify JWT from cookie                               │
│         │                                                        │
│         ├── Valid ──► Inject headers (x-user-*) ──► Next()      │
│         │                                                        │
│         └── Invalid ──► 401 Unauthorized                         │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  Route Handler (Server)                                     ││
│  │  Can call getAuthenticatedUser() or verifyInHandler()       ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
```

## 8. Segurança

| Medida | Implementação |
|--------|---------------|
| XSS Prevention | HttpOnly cookie (inacessível via JS) |
| CSRF Prevention | SameSite=lax + verificações server-side |
| Algorithm Confusion | Usar `jose` com alg fixo (HS256) |
| Secret Strength | 32+ bytes aleatórios |
| Token Expiry | 24h hard limit |
| Error Messages | Generic "credenciais inválidas" (não revelar se email existe) |
