**Milestone:** v0.0.3  
**Created:** 2026-05-28  
**Stack:** Next.js 16 + Vitest + Supertest (via fetch) + Prisma

---

## 1. Contexto

### 1.1 Projeto
- **Nome:** cabala-dos-caminhos
- **Stack:** Next.js 16 App Router, Prisma ORM, PostgreSQL
- **APIs existentes:** 14 rotas de API

### 1.2 Objetivo
Adicionar testes de integração para validar o comportamento das APIs REST existentes.

### 1.3 Escopo
- ✅ Testes para APIs de autenticação (login, logout)
- ✅ Testes para APIs públicas (astrologia, numerologia, odus, ciclos)
- ✅ Testes para APIs protegidas (chat, credits)
- ✅ Testes para middleware JWT
- ❌ Testes E2E (Playwright) - futuro
- ❌ Testes de performance/load - futuro

---

## 2. Stack de Testes

| Ferramenta | Uso |
|------------|-----|
| **Vitest** | Test runner (já existe) |
| **@testing-library/react** | Para testes de componentes (já existe) |
| **msw (Mock Service Worker)** | Mockar requisições externas |
| **node:test** ou **Vitest** | Testes de API com `fetch` nativo |

### Decisão: Testes com fetch nativo
- Não usar Supertest (funciona mal com Next.js App Router)
- Usar servidor de teste com `fetch` requests
- Mockar Prisma com `prismock`

## 5. Test Cases

### 5.1 POST /api/auth/login

| Case | Input | Expected |
|------|-------|----------|
| Login válido | email + senha corretos | 200 + cookie |
| Login inválido | email ou senha incorretos | 401 + error |
| Campos faltando | sem email | 400 + error |
| Campos faltando | sem senha | 400 + error |

### 5.2 POST /api/auth/logout

| Case | Input | Expected |
|------|-------|----------|
| Logout normal | com session | 200 + cookie cleared |

### 5.3 GET /api/creditos (protegido)

| Case | Input | Expected |
|------|-------|----------|
| Com token | JWT válido | 200 + saldo |
| Sem token | sem cookie | 401 Unauthorized |
| Token inválido | JWT corrompido | 401 Unauthorized |

### 5.4 POST /api/astrologia/mapa-natal (público)

| Case | Input | Expected |
|------|-------|----------|
| Dados válidos | data + hora + local | 200 + mapa natal |
| Dados incompletos | sem hora | 400 + error |
