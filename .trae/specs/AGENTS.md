# Specs DOX

## Purpose
Especificações de versão do projeto Akasha.

## Ownership
- `akasha-vX.Y.Z/`: Cada versão tem sua spec
  - `spec.md`: Why, What, Impact
  - `tasks.md`: Lista de tarefas priorizadas
  - `checklist.md`: Verificação de completion

## Local Contracts
- Specs refletem estado planejado, não histórico
- Tags Git marcam releases correspondentes
- Ciclos de memória registram execução

## Work Guidance
- Specs são criados no início de ciclos
- Tasks executadas em ordem de prioridade
- Checklist preenchido ao completar cada item

## Verification
- `pnpm test:run` ao final de cada task
- Revisão de checklist antes de fechar spec
