# App Spec — Sistema Akasha

## 1. Missão (NORTE ABSOLUTO)

Construir um **sistema moderno de tecnologia espiritual** que sintetize
**5 tradições ancestrais** em uma linguagem unificada, **à maneira do
Human Design e do Gene Keys** — sistemas modernos que tomaram múltiplas
tradições e produziram uma **engenharia própria**, não uma colagem.

**O Sistema Akasha NÃO é a soma das 5 tradições. É uma síntese nova
que emerge da pesquisa profunda e do design intencional.**

A inteligência mora no cruzamento. A beleza mora na coerência. O
produto mora na experiência do usuário.

## 2. Os 5 Pilares

| # | Pilar | Domínio | Sistema moderno similar |
|---|-------|---------|------------------------|
| 1 | **Numerologia Cabalística** | Identidade, propósito, ciclos | — (origem única) |
| 2 | **Astrologia** | Céu, trânsitos, tempo | Co-Star, The Pattern, CHANI |
| 3 | **Numerologia Tântrica** | Anatomia sutil, 11 corpos | Gene Keys (estrutura) |
| 4 | **Odu de Nascimento** | Ori, ancestralidade, terra | — (origem única) |
| 5 | **I Ching (64 hexagramas)** | Mutação, jornada, ciclos | **Gene Keys** (mapping) |

> **I Ching é a peça central que falta no sistema atual** — Gene Keys já
> demonstrou como 64 hexagramas podem ser reinterpretados como jornada
> humana (Shadow → Gift → Siddhi). Esta é a **referência primária de
> design de síntese**.

## 3. Sistemas Modernos a Estudar (referências)

**Obrigatórios (estudo profundo):**
- **Gene Keys** — Richard Rudd. 64 hexagramas do I Ching como jornada
  Shadow/Gift/Siddhi. Hologenetic Profile. Sequence of Pearls. Genius
  vs. Disease. **ESTE É O MODELO PRIMÁRIO DE SÍNTESE.**
- **Human Design** — Ra Uru Hu. Bodygraph: 9 centros, 64 portas (de I
  Ching), 36 canais, 192 linhas. Tipo/estratégia/autoridade.
  **ESTE É O MODELO DE UX + VISUALIZAÇÃO.**

**Recomendados (estudo complementar):**
- **MBTI** — 4 funções cognitivas (Jung). Teste de tipo. Enneagrama similar.
- **Enneagrama** — 9 tipos, integrações, níveis. Tradição oral moderna.
- **Co-Star** — UX minimalista para astrologia. Cores, micro-doses.
- **The Pattern** — narrativa personalizada. Linguagem íntima.
- **Astro.com / Astro-Charts** — engine técnico, Ephemeris.
- **Mayan Calendar / Tzolkin** — outro sistema de ciclos.
- **Numerologia.com / Numerologist.com** — tradição numerológica cab.
- **Cabala Clássica / Árvore da Vida** — base cabalística.

**Explorar (surpresa):**
- **Ayurveda + TCM** — síntese medicina/corpo.
- **Sheldrake (Campos Morfogenéticos)** — base conceitual.
- **Cymatics** — som-como-forma. Vale para a parte visual.

## 4. Fases de Desenvolvimento (RESEARCH-FIRST)

### Fase 0 — Research (PRIORIDADE MÁXIMA, antes de qualquer código)

Estudar 8+ sistemas modernos. Para cada um:
- Como eles sintetizaram múltiplas tradições?
- Como fizeram o **mapeamento** (ex: hexagrama → Gene Key)?
- Como desenharam a **UX** (visualização, micro-doses)?
- Como escreveram a **linguagem** (tom, mito, identidade)?
- Qual o **modelo de negócio** (B2C vs B2B)?

Outputs:
- `.autonomous/research/<systemo>.md` — relatório por sistema
- `.autonomous/research/patterns.md` — padrões comuns extraídos
- `.autonomous/research/gaps.md` — o que nenhum sistema cobre

### Fase 1 — Synthesis Design

Após Fase 0: propor **Akasha Core Algorithm** — como os 5 pilares se
cruzam em UMA leitura unificada. **Não é implementação. É design.**

Output: `.autonomous/research/synthesis_v1.md` (e v2, v3… iterações)

### Fase 2 — AI Mentor / Mestre

Definir a personalidade e o system prompt do agente oracular. O mentor
NÃO é um chatbot — é um **guia espiritual** com:
- Voz própria (não generic GPT)
- Conhecimento ancorado no Grimório (RAG)
- Capacidade de síntese (cruza os 5 pilares em tempo real)
- Comportamento ético (sem manipulação, sem determinismo fatal)

Output: `.autonomous/research/mentor_persona.md`

### Fase 3 — UX Architecture

Decidir: mobile-first PWA? Web? App nativo? Multi-camada
(micro-doses)? Como o usuário descobre progressivamente?

Output: `.autonomous/research/ux_architecture.md`

### Fase 4 — Tech Stack

Confirmar/adaptar: Next.js 16, Prisma, Postgres, Redis, OpenAI +
Minimax, Stripe, Vitest. Adicionar o que faltar.

Output: `.autonomous/research/tech_decisions.md`

### Fase 5 — Protótipo de Algoritmo (pseudo-código)

Escrever Akasha Core Algorithm em TS/types ANTES de implementar
features. Validar com casos de teste representativos.

Output: `packages/akasha-core/` (criado na fase 5)

### Fase 6 — Implementação (features)

Somente aqui. Cada feature = 1 item em `feature_list.json` (migrar de
R-NNN research items para F-NNN impl items).

## 5. Regra de Ouro (NÃO NEGOCIÁVEL)

> **Nenhum código de feature antes da Fase 5 (protótipo de algoritmo).**
>
> Fase 0-4 = research + design only. Commits permitidos:
> - Markdown research files
> - Chain-of-thought logs
> - Diagrams (Mermaid, ASCII)
> - Schema prototypes em YAML/JSON (não TS/JS executável)
>
> Commits NÃO permitidos em Fase 0-4:
> - Código de feature em src/, apps/, packages/
> - Mudanças em schema Prisma
> - Componentes UI novos

## 6. AI Mentor — Quem é?

Baseado em estudo da Fase 2:
- **Voz:** terceira pessoa ritualística, com pausas e ritmo.
- **Conhecimento:** todo vindo do Grimório (`grimoire/`) — RAG
  mandatório. LLM **redige**, não **decide**.
- **Comportamento:**
  - Cita fonte do Grimório em cada afirmação
  - Nunca afirma futuro como certo
  - Sempre pergunta antes de diagnosticar
  - Empodera, não vicia
  - Detecta crise emocional e oferece recurso humano
- **Memória:** conversa tem continuidade via `memory/` do projeto.

## 7. Regras Invioláveis (do projeto)

- **IDEIA.md é a fonte** para correspondências. Constantes em
  `src/lib/constants/*.ts` devem coincidir.
- **Typecheck sempre passa:** `pnpm typecheck` antes de commit.
- **Testes passam:** `pnpm test:run` antes de commit.
- **Commits PT-BR:** `tipo: descrição`.
- **Versões:** spec `akasha-vX.Y.Z`; tag git `vX.Y.Z`.
- **AGENTS.md (DOX)** é contrato. Não quebre hierarquia.
- **PROGRESS.md** atualizado a cada fase.
