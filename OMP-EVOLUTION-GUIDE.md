## MEU CONTEXTO:
- A plataforma tem MUITOS widgets que mostram dados sem significado profundo
- O objetivo é agregar conhecimento e autoconhecimento ao usuário
- Tradições: Numerologia Cabalística, Odu Ifá, Astrologia, Tarot, Cabala, Orixás, Chakras, Geometria Sagrada, Ervas
- Estado atual: 237 sprints, 4388 tests, 303+ widgets

## CICLO DE EXECUÇÃO:

### FASE 1: ASSESS
Execute em paralelo:
- read PROGRESS.md
- read memory://root/memory_summary.md
- Execute: git status

Diagnóstico rápido:
- Quantos widgets? Estão agregando valor real ao usuário?
- Correlações implementadas vs. faltando?
- Onde está "caixa preta" de informação sem significado?

### FASE 2: THINK
Use esta cadeia de pensamento:

SITUAÇÃO: O que existe agora sem significado claro?
PROBLEMA: O usuário entende o que está vendo?
POTENCIAL: Qual correlação está faltando entre tradições?
AÇÃO: Criar correlação? Refinar widget? Gerar insight?
META: Aumentar o significado a cada ciclo

### FASE 3: PLAN
Defina 1-3 tarefas para este ciclo:

🔴 CRÍTICO (sempre):
- Correlação espiritual faltando entre tradições
- Widget que confunde mais do que ajuda
- Bug que quebra experiência

🟡 IMPORTANTE (quando não há críticos):
- Nova correlação (3+ tradições)
- Refinamento de insight
- Otimização de engine

🟢 BOM (quando sobrar tempo):
- Test coverage
- Documentação
- Refatoração

### FASE 4: EXECUTE

**REGRA DE OURO:** Use APENAS dados do IDEIA.md (nunca invente correlações)

**REGRA DO USUÁRIO:** Antes de criar/modificar qualquer coisa, pergunte:
- "Este widget/widget ajuda o usuário a se conhecer melhor?"
- "Existe conexão espiritual neste dado?"
- "Posso gerar um insight significativo aqui?"

**TRÊS NÍVEIS DE INSIGHT:**
1. DADO CRU: número, planeta, signo
2. INTERPRETAÇÃO: significado tradicional
3. INSIGHT PESSOAL: correlação + contexto do usuário

**SE WIDGET NÃO TEM SENTIDO:** Remover ou transformar

### FASE 5: VERIFY
```bash
npm run build
npm run lint
npm run test:run
```
Se algum falhar: ARRUME ANTES DE PROSSEGUIR.

### FASE 6: EVOLVE
```bash
git add . && git commit -m "tipo: [descrição]"
```
Tipos: feat, fix, refactor, spiritual-correlation, insight, test

Atualize:
- PROGRESS.md com achievements
- memory se aprendizado significativo

### FASE 7: LOOP
Volte para FASE 1 até não houver mais trabalho significativo.

## QUANDO PARAR

Quando não houver mais:
- Correlações faltando (3+ tradições)
- Widgets sem significado
- Bugs críticos
- Insights quebrados

Gere relatório final do que foi accomplishado.

## CRITÉRIOS DE SUCESSO POR CICLO

✅ Nova correlação espiritual implementada (3+ tradições)
✅ Widget transformado em insight significativo
✅ Build/Lint/Tests passando
✅ Commit descritivo feito

## CADÊNCIA DE TRABALHO

- A cada ciclo: verificar quality metrics
- A cada 10 ciclos: atualizar PROGRESS.md
- A cada 50 ciclos: gerar relatório de evolução

INICIAR CICLO AGORA.
```
