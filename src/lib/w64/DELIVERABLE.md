## 7. Honest concerns

1. **Cigano card `number` field is widened to 1..36** to satisfy `CardRecord.number: NumerologyNumber`. Strict numerology taxonomy (1–9 + 11/22/33 = 12) is preserved as `NUMEROLOGY_NUMBERS` constant for audit purposes. The audit limits `numerologyUsed` to entries present in `NUMEROLOGY_NUMBERS` so coverage count = 12, not 36. This is intentional: card sequential numbers ≠ numerology taxonomy numbers.

2. **`crossHouse` requires the 4 other maps as `externalContext`** (Astrologia, Numerologia, Orixá, Odu engines). When the supplied mesa already has a crossing for the requested house, `crossHouse` returns the cached one. Otherwise it builds a generic synthesis. Engine does NOT import any other w6x modules — caller wires the maps at the registry/factory level.

3. **`auditSacredCoverage` `totals.orixa = 19` (count from `ORIXA_MAPPINGS`)** which includes 3 dual-mappings (cards 7/9/32 each host 2 orixás). Without this, the audit would report 16 instead of 19. The dual mapping preserves 1-to-1 cross-tradition fidelity for the post-game chat synthesis, while keeping each card's primary `orixa` field single-valued for downstream consumer UX.

4. **Planet 'terra' is a known omission.** The 11-planet audit target includes Earth for completeness against the broader Astrologia taxonomy, but classical Cigano mapping covers only the 10 celestial bodies. `auditSacredCoverage` notes this in the missing list with a comment; the engine's `totals.planetas = 10` (the actual covered count). Callers can override.

5. **`interpretSpread` is NOT used by `SPREAD_36_MESA_REAL`** — that path is owned by `interpretMesaReal` (which internally does its own interpretation + house crossings). `interpretSpread` covers 1/3/5/9 sizes. The dispatcher `interpretReading` routes by spread type.

6. **Confidence cap is enforced** at `MAX_CONFIDENCE = 0.99` (not 1.0) per cycle 63 lesson 5 (`boostScoreByCitations` test asserts this). Heuristic confidence is for caller UI affordance, NOT predictive.
